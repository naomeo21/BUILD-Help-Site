Known Issues

The following are known issues.

Published bindings are not displayed in an object page

If the first page of a prototype contains a main object, context is expected. If objects are bound without context, data is displayed in preview mode but is not displayed when published. In this case, we recommend that you not select a main object. Published applications need to have a specific object assigned instead of a collection of objects.

Data binding to a table does not work correctly using the drag and drop function

When you bind to a table using the drag and drop function, you create a self-pointing relationship, which does not work because you need to select the name of the relationship. You need to manually bind the child items of the table using the chain icon.

Horizontal scroll bar does not work correctly

When data is added to an object manually in the Data Editor, the horizontal scroll bar does not work, and you need to use the arrow key to navigate.

Cannot add data to more than one time property type

When you try to manually add data in the Sample Editor to more than one time property type, you are unable to save the data and an error message is returned.

Photo not updated in the banner

When you change your photo in the user settings, the photo is not always updated in the banner.

Popover is not displayed after deletion is undone

When a popover is deleted from a tree view and then the action is undone, the popover is no longer visible on the page.

Incorrect display of an icon added to a tab bar

After an icon tab bar control is added in an object page sub section, it is not displayed correctly in the preview.

Incorrect display of a re-sized hotspot

When a hotspot is created on a blank page and is then re-sized, the hotspot is not displayed in the correct position.

Incorrect image shape

Although the image shape is set to Circle in the header of an object page, the image appears in a oval instead of in a circle.

Images not displayed if imported after data binding

Images are not displayed if they are imported into the Files page after they have been bound to the Image property. Refresh the page to view the image.

No display of date value property for the Date Picker and Date Time Input UI elements

Both the Date Picker and the Date Time Input UI elements in the palette do not display their DateValue property. The user must be able to set the value of this property and see the value in the composer.

Width of radio buttons increases

The size of a radio button automatically increases by 47pixels.

Incorrect display when adding rows of data to an object

When data is added to an object under the Samples tab in the Data Editor where there are more than 25 rows, and a new row is added, the new row is not displayed correctly.

Incorrect display of icon tab bar after deletion of an icon

After an icon is deleted from an icon tab bar, the remaining icons on are not displayed correctly. This problem is solved by refreshing your browser.

Incorrect display on a large Sankey diagram

In a large Sankey diagram, some names are obscured and some nodes are not displayed correctly.

Annotations on impermanent pop-ups and dialogs are displayed incorrectly in feedback results

Annotations added by users on pop-ups on prototype pages in feedback studies are displayed incorrectly in the feedback results.

Annotation added when user scrolls to the bottom of page are displayed incorrectly in feedback results

Annotations are displayed incorrectly in feedback results if the user scrolled to the bottom of the prototype page when they added the annotations.

Some buttons are not displayed in object page header after they have been selected once

Some buttons, such as the back arrow, disappear after they have been selected once. As a workaround, we suggest setting the button to Transparent instead of Back and then setting the icon to sap-icon://nav-back.

Date not displayed in the expected format

When a date type property is added to an object in the data editor, the date is not always displayed in the way the user expects. Dates are displayed in the system format, which is YYYY-MM-DD. Time is displayed as HH:MM:SS AM/PM, and the time zone in UTC. This information is not communicated to the user by the system.

Unexpected behavior when cloning a project

When a user is connected to multiple tenants and clones a project, the project is cloned in the wrong tenant. For example, a user logged in to Tenant A then logs in to Tenant B. He returns to Tenant A and clones a project. The project is cloned in Tenant B instead of in Tenant A.
