Known Issues - Smart Templates

The following known issues concern smart template prototypes.

Boldface value not displayed

In a Smart Template prototype, if you bind a string property to the Boldface Value field of a table column and a numeric property to the Value field of the same column, the boldface value will not be displayed in the preview. As a workaround, in the Data Editor, change the data type of the numeric property to String and re-do the binding. The data displays correctly.

Smart Template Object Page with Freestyle Page

In a Smart Template prototype, when implementing a breakout at the page level (mixing smart template and non-smart template pages), fields from objects linked through a 1:1 relationship from the main object of non-smart template pages won't appear in the published prototype. To work around this issue, avoid one-to-one relationships in your data model. We recommend that you merge the properties of the objects.

Currency value is not displayed

In a Smart Template Object Page, when the Currency property of an object in a 1 - 1 relationship is selected as a second value in a table column from the More Objects list, the currency is not displayed in the table column.

No error message when binding without a main object

You cannot bind by dragging a property to the Value unless you have previously selected a main object is selected. There should be an error message informing you.

No dropdown or value helper when a currency exists
In a Smart Template Object Page, setting the Edit Mode property of a Form element has no effect on the prototype in the following cases:

  • If the property value is bound to a data model property of a date data type, it is displayed using a Date Picker control in edit mode.

  • If both a value and a currency are set for the form element, both are displayed using a standard Input control in edit mode.

Error when editing a Smart Template Object Page

If you try to create a new row in a table by selecting the + button in edit mode in a Smart Template Object Page, the following error message is displayed: No Matching items found.

Back navigation does not work with self-referencing relationships in a Data Model with Smart Template Object Pages

A blank page is displayed in navigation back from a Smart Template Object Page to the preceding Smart Template Object Page. This issue can be resolved by refreshing the browser.

Failed navigation among table rows in Smart Template Object Pages

After navigation among several smart template object page table rows, a blank page is displayed.

Incorrect display of Smart Template List Reports and Smart Template Object Pages across devices

Although the responsiveness property of columns and action buttons is set correctly in Smart Template List Reports and Smart Template Object Pages, the columns and buttons are always visible regardless of the size of the screen.

Dropdown List and Value Help dialogs appear as choices for the Edit Mode with a date property type

When a value with a date property type is selected in a form element, only Standard Input should be available as a choice of edit mode. Other options do not work in Edit Mode because a calendar appears in the editor.

KPI decimal value is truncated

In a Smart Template Object Page, when a numeric type property is bound to a KPI but a currency is not set, the decimal is not displayed in the preview pane. The decimal value should be displayed whether or not a currency value has been selected.

Binding two different form elements to the same value returns a blank page

BUILD allows you to bind two different form elements to the same value, however this returns a blank page. Smart Templates do not support binding to elements to the same value. Once a field is bound, it should not be able to be reused.

Image not displayed in Smart Template Object Page header

Binding the image from the Smart Template Object Page header only works to properties of the main object of the object page. While BUILD does allow you to bind to properties of objects linked via a one-to-one relationship, the binding does not work.
